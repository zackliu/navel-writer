# 常识与逻辑

小说设定上最重要的是维护常识与逻辑，无论背景是否架空，都要从读者角度来想，他们会愿意看到一个架空的设定，但是不是奇怪的运行和人物逻辑

逻辑：因果可推演，前提不变则结果不变。
常识：注意力预算与语域一致——在该场景压力等级下，角色只会做与当前最高目标直接相关的事，并使用该世界仍在运作的那套语言。

逻辑和常识正常 = 场景压力等级一致 + 世界仍在运作的东西一致 + 角色注意力预算一致

# 输出格式约束（写 brief 时必须包含）

- 本章事件核（一句话：谁在什么场景做了什么 → 当场后果是什么）
- 本章局面变形点（具体是什么变了）
- 本章读者回报（一句话：读者这一章“吃到的一口”是什么）
- 回报兑现点（发生在哪个场景、谁对谁做了什么、当场结果是什么）
- 本章代价变化（谁付了什么）
- 章末未完成事件（下一章必须处理什么，且已触发）
- 本章主推进手段（用于检查是否与上一章重复）


# 场景写法（少解释，多行动；对话有潜台词）

- 叙事依靠场景与行动：少解释、少总结，不要用“主题宣讲”替代人物挣扎。
- 对话必须有潜台词：角色不把心里话说满，允许尴尬与留白。
- 允许真实的无用动作：让场景有摩擦，而不是每段都“完整、漂亮、解释动机”。


# 争吵与对话
- 争吵与对话部分不能长，把事情讲清了或者产生了效果就可以结束。因为LLM容易把对话写的很奇怪
- 对话与争吵都必须符合常识与逻辑

# 去模型味（长期一致靠“重复与变形”）

主动避免这些“模型味”：

- 每段都过度完整
- 每段都解释动机
- 每段都漂亮
- 频繁抽象总结
- 用金句制造意义

意义必须来自“重复与变形”：

- 同一句话在不同场景里变成不同的刀。


## 失败判定（触发就重写本章事件）

- 本章主要内容是解释、回忆、讨论、记录，而不是做事并付代价。
- 本章没有“读者回报”，或回报无法落到一个具体场面。
- 本章结尾没有“正在逼近并已触发的麻烦”，只有“决定继续”。
- 本章和上一章靠同一种方式推进（连续几章都靠同一道具/流程/同一种对抗）。
