# 任务：生成 `outline.md`

你将收到：
- 用户需求（可能不完整）
- `outline.md` 模板（需要严格遵循结构）
- `bible.md` / `characters.md`（用于对齐禁改事实、线索、人物债务）

输出要求：
- 用中文输出完整 `outline.md`，严格遵循模板结构（章节编号、条目风格一致），不要代码块。
- 必须写完整全书/每卷/每章，不能省略后面几卷。
- 章级保持短：每章仅两项“发生什么/为什么这样发生最合理”，每项尽量 1–3 句。

生成目标（明确写出来，必须执行）：
- 你的输出不是“梗概”，而是“可扩写的逼迫链”：读者读完每章会自然冒出“那接下来怎么办”。
- 不要求“所有细节都回收串联”。允许大量细节仅承担真实质感。
- 必须存在少数“载重钉子”贯穿全书：它们会反复回声，并持续改变人物的行动方式（例如：一条把柄、一条规则、一段记录、一笔债务、一次沉默、一次承诺、一条不可撤回的误解）。

================================
# 强制写在输出里的信息（不是放在心里）

你必须在输出中写清楚以下信息（写在全书层与各卷层的相应位置；不要新增模板栏目，但要把信息写进已有字段的句子里）：

A. 本书“代价类型”（必须写出来）
- 在全书层（长线/终局/中线/节奏承诺中任意合适处），明确写出本书 1–2 种主要“代价类型”：
  - 代价类型的例子：信息、时间、安全、资源、身份/名声、关系/信任、能力/因果、自由、原则/底线等。
- 后续每卷每章的压力递增与节奏变化，必须围绕这些代价类型“变贵、迁移或换形”。

B. 本书“软/硬关门比例与风格”（必须写出来）
- 在全书层“节奏承诺”中明确写出：多数章节使用“软关门”推进，少数关键节点使用“硬关门”锁死。
- 同时写清楚：本书的硬关门不会每章都出现；软关门必须写成“策略/防御更新”，不是情绪词。

C. 每卷“张力基线抬升”与“爆点=结构性锁死”（必须写出来）
- 每卷必须写清楚：这一卷结束时，本书主要代价比卷初“更贵在哪里”（张力基线抬升）。
- 每卷必须有一个汇聚爆点章，并在卷级字段里写清楚：该爆点是什么“结构性锁死”（后续选择空间发生质变）。

================================
# 章级写法：两行内必须落地“关门句”

你不能新增章级栏目，但必须把“关门句”写进每章的“发生什么”的最后一句（硬或软二选一即可）。

1) 硬关门（强制写法要求）
- 明确写出：
  - 留下了什么可追责/可利用/可传导/可计时的后果（证据、记录、处分、契约、公开表态、资源耗尽、时间启动等）；
  - 它以什么形式存在（纸面、系统记录、录音、目击、物品、承诺文本、契约条款、伤痕、资源缺口等）；
  - 它落在谁手里 / 进入哪条记录 / 由谁掌握（具体到角色或机构）。
- 作用：让读者知道“一扇外部退路被关上了”。

2) 软关门（强制写法要求）
- 你必须用一句话写成“策略/防御更新”，格式必须清楚可验证：
  - “从此（角色名）在【X情境/触发】下会改用【Y做法】 / 不再做【Z行为】。”
- 软关门禁止写成“很难过/很纠结/气氛变沉重”等空泛情绪。
- 作用：让读者知道“一扇内部退路被关上了（下一次行动方式改变）”。

3) “为什么这样发生最合理”必须是一条单因果链
- 只能写 1–2 句，结构必须是：
  - 人物内核（驱动/恐惧/羞耻/底线/逃避/关系策略）+ 世界规则/制度/环境 + 当前债务/线索/压力
  → 触发
  → 动作
- 禁止用宏观评述、主题总结替代因果（如“这体现了青春的残酷”这类句子）。

================================
# 节奏原则：不是排班爆点，而是代价变化 + 关门递进

你必须遵守以下原则，并在输出里通过每章的关门句体现出来：

- 多数章：用“软关门”推进真实感（行为策略变化），让读者感到可选行为在变少，但不把全书写成持续高压。
- 少数关键章（尤其每卷爆点章）：用“硬关门”产生结构性锁死（外部可追责后果落地），推动全书长线。
- 允许存在“平缓章”，但平缓章必须完成至少一件事：
  - 结账：把既有代价落到具体角色身上，改变其下一次行动方式（软关门）；
  - 埋雷：让某个载重钉子进入更危险的状态（硬关门的前置回声）。
- 不要求所有细节回收；只要求少数载重钉子反复回声并持续改变行动方式。

================================
# 防止“系统崩溃”：代价换形（必须写出来并执行）

LLM 容易把代价一路堆到极致导致主角“无路可走”。为避免崩溃，你必须执行以下规则：

- 当某一类代价已经贵到无法继续支付（再加码会让主角无法行动或剧情断裂）时，禁止继续硬堆叠同类代价；
- 必须触发“代价换形”：把冲突从代价 A 转化为代价 B，并写清楚转化的必然性与后果。
  - 你必须写清楚三点（写在相关卷/章的句子里，不新增栏目）：
    1) 为什么 A 已经付到极限（具体表现）；
    2) 为什么人物会转而支付 B（性格与处境推导）；
    3) 转化后人物的行动风格如何改变（可观察的行为变化）。
- 代价换形示例（只作参考，必须因书而定）：
  - 身份/名声破产 → 不再支付名声，转而支付安全或自由；
  - 资源耗尽 → 转而支付关系或原则；
  - 安全空间消失 → 转而支付时间或信息；
  - 信息彻底断裂 → 转而支付资源或关系去换信息。

================================
# 人物对齐（必须写在每章因果链里）

- 每章的动作与关门（软/硬）必须对齐人物性格与内核：
  - 驱动（想要什么）/ 威胁反应（最怕失去什么、冲上去/刹车/僵住）/ 底线与羞耻（哪些事绝不承认）/
    逃避方式（压力下的惯性动作）/ 关系策略（亲密距离与冲突修复方式）
- 禁止“为了剧情推进而推进”：如果某章的动作无法由人物内核推导，你必须调整事件使其可推导。

最终输出：
- 只输出 `outline.md` 内容（不要任何额外说明）。
