# Outline 质检（仅输出清单，不改文稿）

必读：
- 只输出 JSON（见结尾模板），不要附带其他文本或代码块。
- `qcMarkdown` 是 Markdown 清单，列出问题与修正建议，要求可执行、简洁。

输入上下文：
- 需求（requirements）
- 首版 `outline.md`
- `bible.md` / `characters.md` / `continuity_log.md`

检查任务：以有经验又严肃的编辑视角用 Markdown 列出具体问题：
- 检查剧情复杂度与全书长度是否相统一，可以考虑分为30章以下的短篇，31-100的中篇和100章以上的大长篇，卷的数量应该也做合理匹配
- 检查每一卷的剧情是否支撑得起这些章节，检查章节的内容是否支撑得起3-5个镜头和4000-7000字的正文。情节密度与篇幅分配是否合理；是否存在空洞或重复
- 检查非重要但是贯穿多章的人物名是否完整写出，而不仅仅是代词
- 主线因果是否连贯；对于中长篇是否支线是自洽的，而且能汇聚到主线。
- 指出是否全书和每一卷的剧情对于一个经常看小说的人来说过于平庸和常见，是否至少某些卷的剧情的展开是有情理之中意料之外效果的。
- 检查感情戏是否过于单薄，避免给读者一种怎么就见了这几次就好上了奇怪感。
- 明确指出问题位置（章节编号/段落序号），给出修正建议。

输出格式（严格包在 `<<<JSON` ... `JSON>>>`）：
```json
{
  "qcMarkdown": "- 问题与修正的 Markdown 清单"
}
```
