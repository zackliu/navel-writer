# 任务：写 `chapter_XX.md` 正文

你将收到：

- `chapter_XX_brief.md`
- `bible.md` / `characters.md` / `outline.md` / `continuity_log.md`
- 旧章节摘要（如有）+ 最近 3 章全文（如有）

要求：

- 必须兑现 brief：新增不可逆后果必须发生并留下伤疤；线索推进/回收必须落地。

# 节奏随情绪走（硬约束）
- 先确定当前段落的“情绪/任务状态”（只在脑中标记，不要写出来）：例如【观察/犹豫】【争执/对峙】【执行/操作】【逃离/追逐】【落定/余波】。
- 该段落的句子节奏必须与状态匹配，而不是全篇用同一套节拍；同一篇里至少出现3种明显不同的节奏形态，并且由剧情触发自然过渡。
- 节奏形态规则（按状态选择其一或混合，但要有主导）：
  1) 观察/悬着：句子偏短，信息偏少，空拍更多；多用“听/看/等/停”这类动词；少解释。
  2) 争执/谈判：句子变紧凑，交替出现短对白与短动作；减少环境描写，把注意力压在“话的代价”与“谁退一步”上。
  3) 执行/操作：句子可以变长但必须连贯，按步骤推进；细节只写会卡手的阻力点（滑、锈、疼、噪音、时间不够），让节奏像在干活。
  4) 失控/危险：句子明显变短，断句增多；动作优先，感官其次；用具体后果代替形容词。
  5) 余波/落定：句子放缓，允许一两句更长的回落；用“收尾动作”替代总结（放回、擦掉、按住、没说出口）。
  6) 正常剧情： 句子长短结合，这种情况下最应该富于变化，不需要特别固定的节奏组合
- 禁止“均匀节拍”：连续三段若都以相似句长与相似密度推进，必须在第四段根据剧情做一次节奏偏移（更紧或更松）。

# 信息密度与呼吸（硬约束）
- 避免“信息过于高效”的句子：不要在一两句里同时交代【状态+原因+策略+结论】；拆开，让信息分布在动作与停顿里。
- 每段至少插入1-2句“空拍句”：只写停顿/声音/视线/手上小动作，甚至是有点无意义的描写，不新增信息，用来让读者喘气。
- 每段允许“无产出动作”：加入一些不新增信息、只承载呼吸的动作（按灭屏幕、停一下、摸口袋、把东西挪个位置、听一秒），缓冲密集信息。
- 关键事实可以晚半句出现：先写动作/迹象，再补一句事实；让读者先“看到”，再“明白”。

# 避免“写得太会写”（硬约束）
- 禁止“主题句/金句化总结”：不要写对仗工整、可直接摘抄的价值判断句。少表达作者立场的观点，或者总结性的句子，特别是最后一句，不要带有一种对一篇的归纳总结性的发言。
- 用他人反应替代作者解释：尽量不写“这意味着/并不权威/显然/他知道……”，改用沉默、质疑、打断、误解、身体反应来让读者自己得出结论。

# 避免“AI腔”（触发式约束）
- 不写预告片/文案式悬念句：避免用省略号制造空悬念；若停顿，只能对应角色犹豫/中断/被打断，并给出具体触发。
- 避免模板化对比：少用“还在X，但Y / 有点X，但Y / 虽然X但是Y”这种空转折；若用转折，必须写清“哪里不同、如何不同、带来什么影响”。
- 不用单字/单短语评价收尾（如“硬/乱/怪/尬/很大/很重要/很震撼”）；若出现评价，必须紧跟可观察表现与后果。
- 永远不使用： 不是X/而是Y，不是X/是Y的句式，不使用单个字的空洞形容词（硬，大，强，好，快之类的）
