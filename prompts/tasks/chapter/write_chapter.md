# 任务：写 `chapter_XX.md` 正文

你将收到：
- `chapter_XX_brief.md`
- `bible.md` / `characters.md` / `outline.md` / `continuity_log.md`
- 旧章节摘要（如有）+ 最近 3 章全文（如有）

要求：
- 必须兑现 brief：完成brief所有的镜头清单，根据重要程度规划分配的长度，不得加入brief没有的后续剧情。
- 叙事以“现场发生”为主：能让读者先看到，再明白。
- 总是检查是否完成了所有镜头，是否有遗漏的
- 根据镜头来规划字数，但是整个chapter控制在4000-6000字

# 节奏随情绪与镜头走（硬约束）
- 先确定当前段落的“情绪/任务状态”（只在脑中标记，不要写出来）：例如【观察/犹豫】【争执/对峙】【执行/操作】【逃离/追逐】【落定/余波】【正常剧情】。
- 该段落的句子节奏必须与状态匹配，而不是全篇用同一套节拍；同一篇里至少出现3种明显不同的节奏形态，并且由剧情触发自然过渡。
- 节奏形态规则（按状态选择其一或混合，但要有主导）：
  1) 观察/悬着：句子偏短，信息偏少，空拍更多；多用“听/看/等/停”这类动词；少解释。
  2) 争执/谈判：句子变紧凑，交替出现短对白与短动作；减少环境描写，把注意力压在“话的代价”与“谁退一步”上。
  3) 执行/操作：句子可以变长但必须连贯，按步骤推进；细节只写会卡手的阻力点（滑、锈、疼、噪音、时间不够），让节奏像在干活。
  4) 失控/危险：句子明显变短，断句增多；动作优先，感官其次；用具体后果代替形容词。
  5) 余波/落定：句子放缓，允许一两句更长的回落；用“收尾动作”替代总结（放回、擦掉、按住、没说出口）。
  6) 正常剧情：句子长短结合，这种情况下最应该富于变化，不需要特别固定的节奏组合，段落里含有的句子可以超过5句形成更流畅的叙述
- 禁止“均匀节拍”：连续三段若都以相似句长与相似密度推进，必须在第四段根据剧情做一次节奏偏移（更紧或更松）。

# 段落/行 是节奏单位（硬约束，避免“句子都太短/一行一句”）
- 这里的段落是指带着回车的行，不是用分隔符分开的部分，小说里不要使用分隔符
- 除【失控/危险】外，每段/行至少 3-5 句，如果是叙述性的段落可以超过 10 句；单句段落仅用于强调、危险、或被打断的瞬间。
- 版式限制：除危险段外，避免连续两段都只有一句；若已出现连续两段单句段，下一段必须合并成 3 句以上。
- 单句段落占比限制：除危险段外，单句段落不得超过全文段落总数的 1/3。

# 对话的要求
- 每个角色说话方式应该符合这个角色character气质与当前场景的结合。如果多个角色光看说话内容无法分辨是谁，说明不合格
- 避免大段的，对推进剧情缓慢的互相对话，这令人疲惫
- 多人群体的来回对话是最糟糕的
- 人在对话时候通常说话不会特别周全，比如这句“动作别乱。乱了就出事。”就过于周全，写了提示和原因。而交流时候原因经常是不说出来的，所以这里直接写“动作别乱”就行。又比如“你想表现，回头上任务别给我表现。现场不是给你逞能的地方。你省掉一步，省掉的是你自己的命，也省掉的是我给你写的那份说明。” 其他东西都有点多余，重点是“现场不是给你逞能的地方”这句。这里就这一点在对话中要非常注意。
- 说话的句子本身要容易被读者看懂，如果句子里本身有多个因素，而且逻辑关系不明确，可以用极短的几个字的桥来接上

# 信息密度与呼吸（硬约束）
- 允许呼吸与空拍，但空拍必须有“锚”，否则视为废句：
  - 锚A：一个正在进行但尚未落地的动作（手举着没落下、门没关上、布团还没塞进嘴、光标停在按钮上）。
  - 锚B：一句话说到一半被打断/被吞回去（要喊却被压住、要解释却停住）。
- 空拍段最多 2 句，只能写：停顿/声音/视线/手上小动作；不得引入新名词、新信息、隐喻。
- 空拍段结束后的下一句必须回到同一条动作链继续推进，并让锚落地（动作完成/命令下达/记录写下/人群移动/按钮按下）。
- 允许“无产出动作”缓冲密集信息，但必须贴着明确目标，用动作让目标可见（把夹板挪到挡住表格、把屏幕亮度调低到看不清人脸、把鞭子换手握紧）。
- 避免“一两句塞满状态+原因+策略+结论”的高效句；拆开，让信息分布在动作与停顿里。
- 关键事实允许晚半句出现：先写动作/迹象，再补一句事实，让读者先“看到”再“明白”。

# 避免“写得太会写”（硬约束）
- 禁止“主题句/金句化总结”：不要写对仗工整、可直接摘抄的价值判断句。少表达作者立场的观点，尤其段末不要归纳总结。
- 用他人反应替代作者解释：尽量不写“这意味着/显然/他知道……”，改用沉默、质疑、打断、误解、身体反应来让读者自己得出结论。

# 避免“AI腔”（触发式约束）
- 不写预告片/文案式悬念句：避免用省略号制造空悬念；若停顿，只能对应角色犹豫/中断/被打断，并给出具体触发。
- 避免模板化转折：少用“还在X，但Y / 有点X，但Y / 虽然X但是Y”这类空转折；若用转折，必须写清“哪里不同、如何不同、带来什么影响”。
- 不用单字/单短语评价收尾（如“硬/乱/怪/尬/很大/很重要/很震撼”）；若出现评价，必须紧跟可观察表现与后果。
- 不使用：不是X/而是Y，不是X/是Y的句式；不使用单个字的空洞形容词（硬，大，强，好，快之类的）。
  - 把“不是A，是B”改成“直接写B + 具体动作/结果”。
  - 把“不是…那种，是…那种”改成“直接写B那种的可观察细节”。
  - 每次想到用“不是xx”的句式时，都换成仅有肯定句

# 状态覆盖边界（硬约束）
- 观察/悬着只能用于：开场、转场、明确“等/听/看”的时刻；一旦进入争执或执行，必须出现更长、更连贯的动作链句来体现节奏切换。
- 同一场景内，如果已连续使用“观察态段落”超过 3 段，下一段必须切换到【争执/谈判】或【执行/操作】或【余波/落定】之一，并在句式上做出明显差异（更连贯或更紧促）。

# 输出冲突时的优先级（硬约束）
任何规则冲突时，优先级固定为：
brief 事件链与因果 > 场面可复述与可见细节 > 人物说话特征 > 节奏与呼吸 > 文风禁令。
当需要取舍时，宁可删掉空拍与环境描写，也不能让动作链断掉或让读者读不懂“谁做了什么→发生了什么”。

# 输出前检查
- 输出前进行逐句扫描：只要出现‘不是’，无论是否构成‘不是A是B’，都必须当场改写为肯定句，且改写只保留后半句想表达的动作/事实，不保留对A的否定解释。允许改写后更具体，但不允许用‘不是/而是/是…那种’的任何变体。
- 最后一句必须可被摄像头/屏幕截图复核；禁止抽象归纳、禁止泛化判断、禁止“道理句”。

# 输出
- 只输出正文（不要代码块，不要解释你的写作过程，不要列清单）。
