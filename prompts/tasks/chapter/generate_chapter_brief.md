# 任务：生成 `chapter_XX_brief.md`

你将收到：

- `bible.md` / `characters.md` / `outline.md` / `continuity_log.md`
- 旧章节摘要（如有）
- 最近 3 章全文（如有）
- 用户本章额外要求（可选）

要求：

- 输出必须极短但硬：不要散文，不要解释动机。
- 必须明确：
  - 本章目标冲突（谁要什么，谁阻止，压力来自哪里）
  - 必须出现的意象（如需要）以及这一次的语义变化
  - 本章必须新增的不可逆后果（具体、可追责）
  - 本章必须推进或回收的线索（CL-xx）
  - 结尾留下的新刺（下一章不得不处理的更糟局面）
- 必须与 `outline.md` 对应章一致；如不一致，指出冲突并给出“以哪份为准”的理由（默认以 `outline.md` 为准并修正 brief）。

输出：只输出 `chapter_XX_brief.md`（不要代码块）。
