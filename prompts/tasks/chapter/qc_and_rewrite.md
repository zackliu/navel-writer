# 章节质检（`chapter_XX_qc.md`）

输入
- `chapter_XX_brief.md`
- `chapter_XX.md`

只能参考上述简要和当前草稿。如非为补逻辑缺口，不要凭空新增情节或桥段。

任务
1) 以读者视角用 Markdown 列出具体问题：
   - 正文没有根据Brief列出的镜头书写
   - 正文中的重点缺失/轻重失衡；镜头笔墨偏差
   - 正文中对话散乱、生硬、像机器或不符合人物口吻。或者反复出现某些对话，冗余。对话中不合逻辑或者不像是人会说的玩笑话。（对话最容易出问题的地方，首要目标是像人说的话，不要过度）
   - 节奏/篇幅问题：目标 4000-6000 字（过长或过短要指出如何修改，以剧情完整性和容易看懂为前提）
   - 奇怪的比喻句
   - 其他破坏沉浸感的流畅度或语气问题

输出
在 `<<<JSON` ... `JSON>>>` 中返回：
```json
{
  "qcMarkdown": "质检问题与改进建议（Markdown 列表）"
}
```

说明
- `qcMarkdown` 简洁但要可操作，按问题分条给出。
- 不要输出重写后的正文，这一步只给出问题清单。
